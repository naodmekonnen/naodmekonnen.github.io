Weekly Blog Sprint 3

What is pseudocode?

Pseudocode describes the steps that should be taken to solve a problem. It outlines the sequence of events that will be executed when a program runs. Ideally pseudocode should be language agnostic , but will detail the steps in plain language. Pseudocode is the blueprint of the program that will be built. Pseudocode is not restricted by any syntax rules but must be clear and concise.

How does pseudocode help with programming?

Pseudocode helps to work through the logic of the problem before any code is written. It helps to create an algorithm that is easier to understand and follow. Then the code can be written based on this logic. Pseudocode also helps to communicate ideas between programmers and other non-technical stakeholders. It helps to make sure that everyone involved is on the same page and that the code being written will match the design specifications. Pseudocode supports the idea of iterative refinement. Starting with the high-level design, followed by detailed pseudocode and then the source code. This will help catch any potential errors and could save a great deal of time. Iterating through these steps will help in creating clean and efficient code inline with the D.R.Y principle.
